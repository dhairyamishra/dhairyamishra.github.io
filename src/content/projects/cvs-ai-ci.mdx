## Overview

At CVS Health, I built a closed-loop alt-text pipeline for accessibility quality. The system generated candidate captions with a Hugging Face vision-language model, evaluated them with an LLM judge prompted for CVS brand framing, and wrote back only captions that beat the existing alt text.

The current resume frames this as a production-oriented accessibility and applied-AI system: it improved catalog coverage by **18%** while preserving a review boundary around brand fit and caption quality.

## What I Owned

- Built the transformer-based caption generation pipeline.
- Designed the LLM-judge step for comparing generated captions against existing alt text.
- Framed evaluation around CVS brand fit and accessibility quality rather than generic image captioning.
- Connected the workflow to production-grade accessibility automation.

## Hard Problems Solved

- **Avoid blind replacement**: the system only wrote back captions that improved on the existing alt text.
- **Balance accessibility and brand voice**: captions needed to be useful, compliant, and consistent with CVS framing.
- **Turn model output into workflow output**: the value came from closing the loop between generation, evaluation, and controlled update behavior.

## Impact

- Raised catalog coverage by **18%**.
- Created a repeatable applied-AI workflow for accessibility content quality.
- Demonstrated a practical pattern for using VLM generation with an LLM quality gate.

## Why It Matters

This project shows applied ML in a real enterprise workflow: model generation, policy-aware evaluation, and production update behavior all had to work together.

## Tech Stack

- **AI/ML**: Hugging Face vision-language models, LLM-as-judge evaluation
- **Accessibility**: Alt-text quality automation, WCAG-oriented review
- **Backend**: Python
- **Delivery**: Enterprise production pipeline integration
