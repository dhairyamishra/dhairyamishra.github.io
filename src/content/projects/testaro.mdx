## Overview

Testaro is an open-source NPM package for ensemble web accessibility testing. The public arXiv record describes it as checking compliance with about **650 rules** across **8 tools**: alfa, Axe, Equal Access, HTML CodeSniffer, Nu Html Checker, QualWeb, Testaro, and WAVE.

My current public profile describes this as CVS Health accessibility-tooling work connected to the 25th ACM SIGACCESS ASSETS conference. The strongest supported portfolio signal is the engineering pattern: unify several noisy accessibility-testing tools into one reportable workflow that teams can operationalize.

## What I Owned

- Contributed to accessibility tooling and reporting work around Testaro at CVS Health.
- Helped connect multi-tool accessibility findings to practical engineering workflows.
- Worked with Node.js, Playwright, axe-core, and WCAG/ADA-oriented accessibility evaluation.

## Hard Problems Solved

- **Normalize fragmented outputs**: accessibility tools disagree in structure, coverage, terminology, and severity. A usable system needs one reporting layer.
- **Make results actionable**: raw findings are not enough; engineers need reports that can support review, remediation, and repeated testing.
- **Shift accessibility earlier**: ensemble testing is most valuable when it fits development workflows instead of remaining a one-off audit activity.

## Key Features

- **Multi-Tool Aggregation**: Combines checks from several accessibility-testing engines.
- **Unified Reporting**: Produces standardized reports over heterogeneous tool output.
- **Browser-Based Testing**: Uses automated browser execution for repeatable testing.
- **Compliance Coverage**: Evaluates a large rule set related to web accessibility compliance.

## Why It Matters

Testaro shows platform-style accessibility engineering: consolidate fragmented testing signals, make them repeatable, and turn them into an output surface that teams can act on.

## Tech Stack

- **Runtime**: Node.js
- **Testing**: Playwright, axe-core
- **Domain**: WCAG/ADA accessibility evaluation
- **Reporting**: Unified ensemble reports
