## Overview

Built an internal Retrieval-Augmented Generation support assistant at CVS Health that integrates directly with Slack to provide self-serve troubleshooting for engineering teams. The system combines **OpenAI generation**, **ChromaDB retrieval**, enterprise governance controls, and deployment on CVS-managed **GCP infrastructure** via Terraform and ArgoCD.

The important part of this work is not simply “used RAG.” The real value was turning internal knowledge into an operational support surface that reduced manual ticket load while fitting enterprise platform constraints around deployment, governance, and trust.

## What I Owned

- Built the internal support-assistant workflow and retrieval stack
- Integrated the assistant into Slack where teams already worked
- Shaped the retrieval + generation loop around internal documentation and past resolutions
- Deployed the system through enterprise GCP infrastructure and CI/CD controls

## Hard Problems Solved

- **Knowledge access in the flow of work**: instead of another portal, the assistant had to live inside Slack and answer in a way that was immediately useful during troubleshooting
- **Grounded generation**: answers had to be tied to retrieved internal sources rather than free-form model responses
- **Enterprise controls**: the product needed governance, auditability, and safe deployment characteristics appropriate for internal engineering use

## Key Features

- **Slack Integration**: Native Slack bot for seamless team interaction
- **RAG Architecture**: ChromaDB vector store for document retrieval, OpenAI for answer generation
- **Self-Serve Troubleshooting**: Teams can resolve common issues without filing tickets
- **Context-Aware Responses**: Retrieves relevant documentation and past solutions
- **Secured Governance**: Enterprise auth, request audit logs, and content guardrails

## Impact

- **20% reduction** in manual ticket resolution time
- Accelerated onboarding for new team members
- Reduced load on platform support teams
- Improved knowledge discoverability across the engineering organization

## Why It Matters

This project shows applied ML system design in a real enterprise environment: retrieval quality, integration ergonomics, governance, and platform deployment all matter at the same time.

## Tech Stack

- **AI/ML**: OpenAI API, ChromaDB (vector database), RAG pipeline
- **Integration**: Slack Bot API
- **Backend**: Python, FastAPI
- **Infrastructure**: GCP (CVS Enterprise clusters), Terraform, ArgoCD
- **Data**: Internal documentation corpus, runbook collection
